config.toml

使用者層級設定位於 ~/.codex/config.toml。你也可以在 .codex/config.toml 加入專案範圍的覆寫設定。Codex 只在信任該專案時才載入專案範圍的設定檔。

關於沙箱與審核相關的設定鍵（approval_policy、sandbox_mode、 sandbox_workspace_write.*），請搭配 Sandbox and approvals、 Protected paths in writable roots 以及 Network access 等文件閱讀。

JSON Schema 與 IDE 自動補全

你可以在 這裡 找到 config.toml 的最新 JSON Schema。

若要在 VS Code 或 Cursor 中編輯 config.toml 時獲得自動補全與診斷功能，可安裝 Even Better TOML 擴充套件，並在 config.toml 頂部加入以下這行：

#:schema https://developers.openai.com/codex/config-schema.json

requirements.toml

requirements.toml 是管理員強制執行的設定檔，用於限制使用者無法覆寫的安全敏感設定。 若需詳細說明、存放位置與範例，請參閱 Admin-enforced requirements 文件。

ChatGPT Business 與 Enterprise 使用者的 Codex 也可套用雲端抓取的 requirements。 在 [features] 中使用與 config.toml 相同的 canonical 鍵名來釘選功能旗標；未指定的鍵不受限制。

i 在 requirements.toml 中使用 [features]，以 config.toml 的 [features] 表所使用的 canonical 鍵名來釘選功能旗標。省略的鍵不受限制。
